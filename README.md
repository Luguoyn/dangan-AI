# 弹丸论破风格学级裁判游戏

> Godot 4.x (Forward+) · GDScript · 剧本驱动框架

基于《弹丸论破》系列（Danganronpa）风格的学级裁判游戏框架。剧本与游戏逻辑完全解耦——编剧在 JSON 中编写剧情，引擎逐条解释执行。

## 功能演示

- **2D 日常篇**：玩家移动、NPC 对话（3角色独立剧本）、调查点证据收集、选项分支、屏幕特效
- **3D 学级裁判场**：CSG 圆形法庭、16 站位、自动分配角色、摄像机运镜
- **无休止议论**：句中矛盾词高亮（真/假）、轮流发言循环、Tab 言弹选择、BREAK 碎裂特效
- **反论**：判定线斩击、Perfect/Great/Good 连击
- **拼字三种模式**：字母环旋转、汉字偏旁、词汇排序
- **高潮再现推理**：漫画分格图块放置
- **剧本系统**：27 个 JSON 指令处理器、条件跳转、标记位、好感度
- **调试控制台**：`~` 键打开，hp/all_evidence/affection/skill 等命令

## 快速开始

1. 用 Godot 4.x 打开项目
2. 按 **F5** 运行
3. 进入 2D 场景 → WASD 移动，E 对话，T 进入裁判场
4. 裁判场自动播放剧情，D/R/H/C/B 触发各小游戏

详细操作见 [用户手册](用户手册.md)。

## 项目结构

```
res://
├── scripts/
│   ├── autoload/       ← 13 个全局单例（EventBus, ScriptInterpreter, GameManager...）
│   ├── courtroom/      ← 裁判场场景 + 摄像机 + 站位
│   ├── managers/       ← 10 个管理器
│   ├── minigames/      ← 小游戏 UI（议论/反论/拼字/高潮再现）
│   ├── resources/      ← Resource 类 + 玩法配置 JSON
│   └── ui/             ← 对话框/选项/特效/HP/技能面板
├── scenes/             ← main.tscn, 3d/courtroom.tscn
├── resources/          ← 角色数据 + 议论/反论/拼字配置
├── story/              ← 剧本 JSON（主剧本 + 测试 + 演示）
└── project.godot       ← 13 Autoload, 11 InputMap, Forward+
```

## 技术栈

- **引擎**：Godot 4.x (Forward+)
- **语言**：GDScript
- **渲染**：2D CanvasLayer + 3D CSG 原型几何体
- **剧本**：JSON 驱动，27 个指令处理器
- **架构**：13 个 Autoload 全局单例 + EventBus 信号总线
